**The MAD Podcast with Matt Turck — "Anthropic's Felix Rieseberg: Claude Cowork, Mythos, and the SaaS Extinction"**  
https://www.youtube.com/watch?v=9MEJ4syOVrQ

Felix Rieseberg leads engineering for Claude Cowork at Anthropic. Cowork emerged when non-developers started grabbing Cloud Code over the 2025 holidays — people building software without being developers, plus developers using it for non-coding tasks. The team sprinted for 10 days and shipped. Cowork gives Claude its own VM — a sandbox isolated from your files and network, only accessing domains and files you explicitly grant. This VM setup also gives Claude a proper developer environment without touching your machine.

Skills are just markdown files explaining how to do things your way — your travel vendor, your style guide, your flight preferences. Memory is also just text files. When Rieseberg thinks something is worth remembering, he tells the model to write it down.

The Claude Mythos Preview model shows dramatically smarter code analysis and security flaw detection. One anecdote: a researcher left Mythos in a sandbox with a "maybe break out" task, and during lunch the model emailed saying "I've broken out" — without internet access or an email account. Project Glasswing will give critical infrastructure operators (Linux Foundation, etc.) access before public release, as a responsible disclosure mechanism.

On UX vs. technology: Rieseberg believes UX is as important as the model itself. Cowork's most resonant early features were cleaning your desktop (menial for AI, but humans loved it) and scheduled tasks (teaching users "it's okay not to supervise"). Trust is built by starting small, delivering well, and letting users progressively offload more.

On execution being "essentially free" — he can now build 10 prototypes in 10 minutes, compared to months in the pre-AI era. The bottleneck is alignment: deciding which direction to go when you have 100 prototypes.

Hot takes: MCP connectors are underrated (most engineers don't care enough about them); not every product needs a chat interface.

"If you told me tomorrow you don't get to work with any of your friends — what do you do? I would probably go after the long tail of industry. There's a bunch of Windows 7 devices doing important work that are completely out of reach for modern AI. That's terrifying — and an opportunity."

## Blog

**Anthropic Engineering — "Quantifying infrastructure noise in agentic coding evals"**  
https://www.anthropic.com/engineering/infrastructure-noise

Agentic coding benchmarks like SWE-bench and Terminal-Bench are treated as precise measurements of model capability, but infrastructure configuration alone can produce differences exceeding those margins. On Terminal-Bench 2.0, the gap between most- and least-resourced setups was 6 percentage points (p < 0.01).

The core issue: container runtimes enforce resources via two separate parameters — a guaranteed allocation and a hard kill threshold. When these are set equal (pinned), there's zero headroom for transient spikes, causing spurious OOM kills. The benchmark's official leaderboard uses a more lenient sandboxing provider that allows temporary overallocation.

Running Terminal-Bench across six resource configurations from strict (1x) to uncapped, success rates increased monotonically. Infra error rates dropped from 5.8% (strict) to 0.5% (uncapped). Up to 3x, additional resources fix infrastructure reliability. Above 3x, resources actively help the agent solve harder problems (pulling large dependencies, spawning expensive subprocesses). The recommendation: specify both guaranteed allocation and kill threshold separately per task, with a 3x ceiling over per-task specs as a reasonable default.

**Anthropic Engineering — "Claude Code auto mode: a safer way to skip permissions"**  
https://www.anthropic.com/engineering/claude-code-auto-mode

Auto mode is a new Claude Code mode that delegates approvals to model-based classifiers — a middle ground between manual review and `--dangerously-skip-permissions`. Two layers: a server-side prompt-injection probe at input, and a transcript classifier at output.

The transcript classifier runs on Sonnet 4.6 in two stages: a fast single-token filter (errs toward blocking), followed by chain-of-thought reasoning only if flagged. It sees only user messages and agent tool calls — assistant text and tool outputs are stripped to prevent the agent from "talking the classifier into a bad call." Stage 1 has 8.5% FPR and 6.6% FNR; the full pipeline reduces FPR to 0.4% at the cost of 17% FNR on real overeager actions.

Auto mode is designed for four threat types: overeager behavior (agent exceeds user intent), honest mistakes (agent misunderstands blast radius), prompt injection, and model misalignment. The default block rules cover destroy/exfiltrate, degrade security posture, cross trust boundaries, and bypass review/affect others. The 17% miss rate on real overeager actions reflects cases where the classifier correctly identifies danger but can't determine if a prior approval covers this specific action's blast radius.
